# Чеклист для тестирования

## Быстрая проверка функциональности

### Подготовка
- [ ] Убедитесь, что `OPENROUTER_API_KEY` настроен в `.env.local`
- [ ] Запустите сервер разработки: `npm run dev` или `pnpm dev`
- [ ] Откройте приложение в браузере: http://localhost:3000

### Базовое тестирование

#### 1. Парсинг статьи
- [ ] Введите валидный URL статьи (например, https://example.com/article)
- [ ] Нажмите "Парсить статью"
- [ ] Проверьте, что результат отображается в блоке результатов
- [ ] Проверьте, что данные содержат title, date, content

#### 2. Тестирование кнопки "О чем статья?" (Summary)
- [ ] После парсинга нажмите "О чем статья?"
- [ ] Проверьте, что отображается индикатор загрузки
- [ ] Проверьте, что кнопка показывает "Обработка..."
- [ ] Дождитесь результата
- [ ] Проверьте, что результат отображается корректно
- [ ] Нажмите кнопку еще раз - результат должен показаться мгновенно (из кэша)
- [ ] Проверьте, что на кнопке появилась галочка ✓ (индикация кэша)

#### 3. Тестирование кнопки "Тезисы" (Thesis)
- [ ] Нажмите "Тезисы"
- [ ] Проверьте индикатор загрузки
- [ ] Проверьте, что результат - нумерованный список
- [ ] Проверьте кэширование (повторное нажатие)

#### 4. Тестирование кнопки "Пост для Telegram" (Telegram)
- [ ] Нажмите "Пост для Telegram"
- [ ] Проверьте индикатор загрузки
- [ ] Проверьте, что результат содержит эмодзи
- [ ] Проверьте кэширование (повторное нажатие)

### Тестирование обработки ошибок

#### 5. Валидация входных данных
- [ ] Попробуйте нажать кнопку AI-обработки без парсинга статьи
- [ ] Проверьте, что появляется alert с предупреждением
- [ ] Проверьте, что кнопки неактивны до парсинга

#### 6. Обработка ошибок API
- [ ] Временно удалите или измените `OPENROUTER_API_KEY` в `.env.local`
- [ ] Перезапустите сервер
- [ ] Попробуйте выполнить AI-обработку
- [ ] Проверьте, что ошибка отображается в красном блоке
- [ ] Восстановите правильный API ключ

#### 7. Тестирование с невалидным URL
- [ ] Введите невалидный URL
- [ ] Нажмите "Парсить статью"
- [ ] Проверьте, что ошибка отображается корректно

### Тестирование производительности

#### 8. Тестирование с разными размерами статей
- [ ] Протестируйте с короткой статьей (< 10,000 символов)
- [ ] Протестируйте со средней статьей (10,000 - 50,000 символов)
- [ ] Протестируйте с длинной статьей (> 80,000 символов, если найдете)
- [ ] Для длинных статей проверьте, что обработка происходит по частям
- [ ] Проверьте время ответа (должно быть разумным)

### Тестирование UX

#### 9. Проверка индикаторов загрузки
- [ ] Проверьте, что спиннер отображается во время загрузки
- [ ] Проверьте, что текст под спиннером соответствует действию
- [ ] Проверьте, что все кнопки неактивны во время загрузки
- [ ] Проверьте, что активная кнопка визуально выделена

#### 10. Проверка кэширования
- [ ] Выполните все три типа обработки для одной статьи
- [ ] Проверьте, что на всех кнопках появились галочки ✓
- [ ] Переключайтесь между кнопками - результаты должны показываться мгновенно
- [ ] Распарсите новую статью
- [ ] Проверьте, что кэш очистился (галочки исчезли)

#### 11. Проверка отображения результатов
- [ ] Проверьте, что результаты читаемы
- [ ] Проверьте форматирование текста
- [ ] Проверьте, что ошибки отображаются в красном блоке
- [ ] Проверьте работу в темной теме (если доступна)

### Тестирование граничных случаев

#### 12. Статьи без даты
- [ ] Найдите статью без даты публикации
- [ ] Проверьте, что парсинг работает корректно
- [ ] Проверьте, что AI-обработка работает

#### 13. Статьи с особыми символами
- [ ] Протестируйте статью с эмодзи в заголовке
- [ ] Протестируйте статью с HTML-разметкой
- [ ] Проверьте, что все работает корректно

#### 14. Быстрое переключение между действиями
- [ ] Быстро нажмите несколько кнопок подряд
- [ ] Проверьте, что приложение не ломается
- [ ] Проверьте, что последний запрос обрабатывается корректно

## Критерии успешного тестирования

✅ Все кнопки работают корректно
✅ Ошибки обрабатываются и отображаются
✅ Кэширование работает правильно
✅ Индикаторы загрузки отображаются корректно
✅ Результаты читаемы и правильно отформатированы
✅ Приложение работает стабильно при различных сценариях

## Известные ограничения

- Обработка очень длинных статей (>200,000 символов) может занять значительное время
- Кэш хранится только в памяти браузера (очищается при перезагрузке страницы)
- Параллельная обработка частей длинных статей не реализована (обработка последовательная)
